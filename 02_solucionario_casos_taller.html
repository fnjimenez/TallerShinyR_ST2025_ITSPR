📝 EJERCICIO 1: Primera App Interactiva

Objetivo:

Crear una aplicación básica que calcule el porcentaje de meta alcanzada.

Solución Paso a Paso:

1. Estructura de la Aplicación

library(shiny)
library(bslib)

ui <- page_sidebar(...)
server <- function(input, output, session) {...}
shinyApp(ui, server)

Explicación:

  • library(shiny) - Carga el framework principal
  • library(bslib) - Carga herramientas de diseño moderno
  • ui - Define la interfaz visual
  • server - Define la lógica y cálculos
  • shinyApp() - Conecta UI y server

2. Componentes UI

sidebar = sidebar(
  sliderInput("produccion", "Unidades Producidas:", 
              min = 0, max = 1000, value = 500)
)

Explicación:

  • sliderInput() - Crea control deslizante
  • "produccion" - ID único del input (para usarlo en server)
  • min/max - Rango permitido
  • value = 500 - Valor inicial por defecto

3. Lógica Reactiva

output$porcentaje <- renderText({
  pct <- round((input$produccion / input$meta) * 100, 1)
  paste0(pct, "%")
})

Explicación:

  • renderText() - Función que genera texto dinámico
  • input$produccion - Lee el valor del slider
  • round(..., 1) - Redondea a 1 decimal
  • paste0() - Concatena texto con el símbolo %

📊 EJERCICIO 2: Dashboard de Control de Calidad

Objetivo:

Calcular First Pass Yield (FPY) y visualizar resultados de inspección.

Solución Detallada:

1. Cálculo de FPY

Fórmula:

FPY = (Unidades Buenas / Unidades Inspeccionadas) × 100

Implementación:

fpy_valor <- reactive({
  buenas <- input$inspeccionadas - input$defectuosas
  (buenas / input$inspeccionadas) * 100
})

Ejemplo de Cálculo:

Inspeccionadas: 100
Defectuosas: 5
Buenas: 100 - 5 = 95

FPY = (95 / 100) × 100 = 95%

2. Gráfico de Barras

df <- data.frame(
  categoria = c("Buenas", "Defectuosas"),

ggplot(df, aes(x = categoria, y = cantidad, fill = categoria)) +

🎯 EJERCICIO 3: Calculadora OEE

Objetivo:

Implementar el cálculo completo de OEE con sus tres componentes.

Solución Completa:

1. Componente 1: DISPONIBILIDAD

Fórmula:

Disponibilidad = (Tiempo Planeado - Downtime) / Tiempo Planeado

Ejemplo:

Tiempo Planeado: 480 min (8 horas)
Downtime: 45 min
Tiempo Operativo: 435 min

Disponibilidad = 435 / 480 = 0.906 = 90.6%

Interpretación:

  • 90.6% significa que el equipo estuvo productivo 90.6% del tiempo
  • 9.4% del tiempo se perdió por paros
  • Meta típica: > 90%

2. Componente 2: RENDIMIENTO

Fórmula:

Rendimiento = (Unidades Producidas × Tiempo Ciclo Ideal) / Tiempo Operativo

Ejemplo:

Unidades Producidas: 450
Ciclo Ideal: 1 min/unidad
Tiempo Operativo: 435 min

Rendimiento = (450 × 1) / 435 = 1.034 = 103.4%

⚠️ IMPORTANTE:

  • Si resultado > 100%, ajustar a 100%

  • 100% indica que el ciclo “ideal” no es realista

  • Revisar y ajustar estándar de tiempo

Corrección:

Rendimiento = min(103.4%, 100%) = 100%

3. Componente 3: CALIDAD

Fórmula:

Calidad = Unidades Buenas / Unidades Producidas

Ejemplo:

Producidas: 450
Defectuosas: 20
Buenas: 430

Calidad = 430 / 450 = 0.956 = 95.6%

Interpretación:

  • 95.6% de piezas salieron bien a la primera
  • 4.4% requirieron retrabajo o fueron desechadas
  • Meta World Class: > 99%

4. OEE TOTAL

Fórmula:

OEE = Disponibilidad × Rendimiento × Calidad

Cálculo Final:

OEE = 90.6% × 100% × 95.6% = 86.6%

Clasificación: WORLD CLASS ✅

Tabla de Clasificación OEE:

Rango OEE Clasificación Color Significado
≥ 85% World Class 🟢 Verde Manufactura de clase mundial
65-84% Bueno 🟡 Amarillo Competitivo, con oportunidades
40-64% Regular 🟠 Naranja Requiere mejoras significativas
< 40% Deficiente 🔴 Rojo Problemas críticos

Análisis de Pérdidas:

Pérdidas por Componente:

Pérdida Total = 100% - 86.6% = 13.4%

Desglose:
- Pérdida por Disponibilidad: 9.4%
- Pérdida por Rendimiento: 0% (ya al 100%)
- Pérdida por Calidad: 4.4%

Conclusión: Atacar primero la Disponibilidad (9.4%)
           para mayor impacto en OEE

# Dashboard Shiny + ML
library(shiny)
library(randomForest)
library(plotly)

# Cargar modelo pre-entrenado
modelo_falla <- readRDS("modelo_predictivo_robot.rds")

# Datos en tiempo real (simulados para demo)
datos_sensores <- reactive({
  # En producción: leer de API de sensores
  data.frame(
    temperatura = input$temp_actual,
    vibracion_x = input$vib_x,
    vibracion_y = input$vib_y,
    corriente = input$corriente,
    ciclos_mantto = input$ciclos
  )
})

# Predicción
prediccion_falla <- reactive({
  predict(modelo_falla, datos_sensores(), type = "prob")
})

# Visualización
output$gauge_riesgo <- renderPlotly({
  prob <- prediccion_falla()[,2] * 100
  
  plot_ly(
    type = "indicator",
    mode = "gauge+number",
    value = prob,
    gauge = list(
      axis = list(range = list(0, 100)),
      bar = list(color = ifelse(prob > 60, "red", 
                         ifelse(prob > 30, "orange", "green"))),
      steps = list(
        list(range = c(0, 30), color = "lightgreen"),
        list(range = c(30, 60), color = "yellow"),
        list(range = c(60, 100), color = "lightcoral")
      )
    )
  )
})

# Recomendación automática
output$recomendacion <- renderUI({
  prob <- prediccion_falla()[,2]
  
  if(prob > 0.7) {
    div(class = "alert alert-danger",
        h4("⚡ ACCIÓN INMEDIATA REQUERIDA"),
        p("Probabilidad de falla: ", round(prob*100, 1), "%"),
        p("Programar mantenimiento en próximas 24 horas"),
        actionButton("programar", "Programar Mantenimiento")
    )
  } else if(prob > 0.4) {
    div(class = "alert alert-warning",
        h4("⚠️ MONITOREO CERCANO"),
        p("Probabilidad de falla: ", round(prob*100, 1), "%"),
        p("Verificar sensores en próximos 3 días")
    )
  } else {
    div(class = "alert alert-success",
        h4("✅ OPERACIÓN NORMAL"),
        p("Probabilidad de falla: ", round(prob*100, 1), "%"),
        p("Continuar operación estándar")
    )
  }
})

PREGUNTAS FRECUENTES

❓ FAQ Técnicas

P1: ¿Por qué mi OEE bajó después de mejorar la calidad?

R: Esto es normal. Ejemplo:

Antes:
Disponibilidad: 95%
Rendimiento: 90%
Calidad: 90%
OEE = 77.0%

Después (más inspección):
Disponibilidad: 92% (más tiempo inspeccionando)
Rendimiento: 88% (más lento para calidad)
Calidad: 98% (mejor)
OEE = 79.2%

Aunque OEE subió solo 2%, la calidad mejoró 8%
Es un trade-off aceptable a corto plazo

P2: ¿Qué hacer si tengo MTBF bajo pero MTTR bajo también?

R: Situación: Muchas fallas pero reparaciones rápidas

Análisis:
- Problema: Fallas frecuentes (MTBF bajo)
- Positivo: Respuesta rápida (MTTR bajo)

Acción:
1. Mantener equipo de respuesta rápida ✅
2. Enfocarse en PREVENIR fallas
3. Implementar mantenimiento predictivo
4. Analizar causa raíz de fallas recurrentes

Meta: Aumentar MTBF manteniendo MTTR bajo

P3: ¿Cómo interpretar un Pareto plano?

R: Si todas las causas son similares:

Causa A: 22%
Causa B: 20%
Causa C: 18%
Causa D: 17%
Causa E: 23%

Interpretación:
- No hay "pocos vitales"
- Problema sistémico, no puntual
- Requiere enfoque holístico

Acción:
- No atacar causas individuales
- Revisar proceso completo
- Considerar rediseño
