📝 EJERCICIO 1: Primera App Interactiva

Objetivo:

Crear una aplicación básica que muestre producción y calcule % de meta.

Paso a Paso:

1. Crear Nuevo Archivo

  • Menú: File → New File → R Script
  • Guardar como: "ejercicio_1_basico.R"

2. Copiar el Código Base

library(shiny)
library(bslib)

ui <- page_sidebar(
  title = "Mi Primera App Industrial",
  
  sidebar = sidebar(
    h4("Control de Producción"),
    sliderInput("produccion", 
                "Unidades Producidas:", 
                min = 0, max = 1000, value = 500),
    sliderInput("meta", 
                "Meta de Producción:", 
                min = 0, max = 1000, value = 800)
  ),
  
  layout_columns(
    value_box(
      title = "Producción Actual",
      value = textOutput("produccion_texto"),
      showcase = bs_icon("gear-fill")
    ),
    value_box(
      title = "% de Meta",
      value = textOutput("porcentaje")
    )
  )
)

server <- function(input, output, session) {
  output$produccion_texto <- renderText({
    paste(input$produccion, "unidades")
  })
  
  output$porcentaje <- renderText({
    pct <- round((input$produccion / input$meta) * 100, 1)
    paste0(pct, "%")
  })
}

shinyApp(ui, server)

🎨 EJERCICIO 2: Dashboard de Calidad

Objetivo:

Crear un dashboard para inspección de calidad con cálculo de First Pass Yield.

Conceptos Nuevos:

  • Inputs numéricos (numericInput)
  • Selectores (selectInput)
  • Gráficos (plotOutput)
  • Condicionales (if/else en render)

Código Completo:

library(shiny)
library(bslib)
library(ggplot2)

ui <- page_sidebar(
  title = "Control de Calidad",
  
  sidebar = sidebar(
    h4("Parámetros"),
    numericInput("inspeccionadas", 
                 "Piezas Inspeccionadas:", 
                 100, min = 1),
    numericInput("defectuosas", 
                 "Piezas Defectuosas:", 
                 5, min = 0),
    selectInput("turno", "Turno:",
                choices = c("Matutino", "Vespertino", "Nocturno"))
  ),
  
  value_box(
    title = "First Pass Yield",
    value = textOutput("fpy")
  ),
  
  card(
    plotOutput("grafico_calidad")
  )
)

server <- function(input, output) {
  
  fpy_valor <- reactive({
    buenas <- input$inspeccionadas - input$defectuosas
    (buenas / input$inspeccionadas) * 100
  })
  
  output$fpy <- renderText({
    paste0(round(fpy_valor(), 2), "%")
  })
  
  output$grafico_calidad <- renderPlot({
    df <- data.frame(
      tipo = c("Buenas", "Defectuosas"),
      cantidad = c(input$inspeccionadas - input$defectuosas, 
                   input$defectuosas)
    )
    
    ggplot(df, aes(x = tipo, y = cantidad, fill = tipo)) +
      geom_bar(stat = "identity") +
      scale_fill_manual(values = c("Buenas" = "#28a745", 
                                   "Defectuosas" = "#dc3545")) +
      theme_minimal() +
      labs(title = paste("Turno:", input$turno))
  })
}

shinyApp(ui, server)

📊 EJERCICIO 3: Calculadora OEE

Objetivo:

Implementar el cálculo completo de OEE con sus 3 componentes.

Fórmulas a Implementar:

Disponibilidad = (Tiempo Planeado - Downtime) / Tiempo Planeado

Rendimiento = (Unidades × Ciclo Ideal) / Tiempo Operativo

Calidad = Unidades Buenas / Unidades Producidas

OEE = Disponibilidad × Rendimiento × Calidad

Interfaz de la App:

Inputs (Sidebar):

  • ⏰ Tiempo Planeado (min)
  • 🛑 Tiempo de Paro (min)
  • 📦 Unidades Producidas
  • ⏱️ Tiempo de Ciclo Ideal (min)
  • ✅ Unidades Buenas

Outputs (Main):

  • 📈 OEE Total (%) + Clasificación
  • 📊 Disponibilidad (%)
  • 📊 Rendimiento (%)
  • 📊 Calidad (%)
  • 📉 Gráfico de componentes
  • 📍 Análisis detallado

Interpretación de Resultados:

📍 Caso de Estudio:

Datos de Ejemplo:

Tiempo Planeado: 480 min (8 horas)
Downtime: 45 min
Unidades Producidas: 450
Ciclo Ideal: 1 min/unidad
Unidades Buenas: 430

Cálculos:

Disponibilidad = (480 - 45) / 480 = 90.6%
Rendimiento = (450 × 1) / 435 = 103.4% → 100% (máx)
Calidad = 430 / 450 = 95.6%

OEE = 90.6% × 100% × 95.6% = 86.6% ✅ World Class!

SOLUCIÓN DE PROBLEMAS

❓ Problemas Comunes y Soluciones

1. Error: “could not find function…”

Causa: Paquete no cargado

Solución:

# Verificar que el paquete esté instalado
if (!require("shiny")) install.packages("shiny")

# Cargar el paquete
library(shiny)

2. La app no se actualiza

Causa: Cambios no guardados o caché

Solución:

# Guardar archivo (Ctrl+S)
# Detener app (Stop button rojo)
# Limpiar workspace
rm(list = ls())
# Volver a ejecutar

3. Error al cargar datos CSV

Causa: Ruta incorrecta o archivo no encontrado

Solución:

# Verificar ruta actual
getwd()

# Listar archivos
list.files()

# Usar ruta relativa correcta
datos <- read.csv("datos/archivo.csv")

# O ruta completa
datos <- read.csv("/cloud/project/datos/archivo.csv")

4. Gráfico no se muestra

Causa: Dimensiones incorrectas o error en el código

Solución:

# Especificar altura del gráfico
plotOutput("mi_grafico", height = "400px")

# Verificar que renderPlot tenga código válido
output$mi_grafico <- renderPlot({
  # Debe retornar un objeto gráfico
  plot(1:10)
})

5. App muy lenta

Causa: Cálculos repetitivos o datos grandes

Solución:

# Usar reactive() para cálculos costosos
datos_procesados <- reactive({
  # Solo se ejecuta cuando cambian los inputs
  procesar_datos(input$parametro)
})

# Limitar actualizaciones
observeEvent(input$actualizar, {
  # Solo actualiza cuando se hace clic
})

6. No puedo publicar

Causa: Cuenta sin permisos o límite alcanzado

Solución:

  • Verificar plan de Posit.Cloud (Free = 25 hrs/mes)
  • Eliminar apps antiguas para liberar espacio
  • Actualizar plan si es necesario

💡 Tips y Mejores Prácticas

Desarrollo:

  1. Comentar el código mientras escribes
  2. Probar frecuentemente cada componente
  3. Usar nombres descriptivos para variables
  4. Modularizar funciones complejas
  5. Guardar versiones regularmente

Diseño:

  1. Menos es más - evita saturar la interfaz
  2. Jerarquía visual - lo importante arriba
  3. Colores consistentes - usa paleta definida
  4. Espaciado adecuado - respira el diseño
  5. Mobile-first - piensa en dispositivos móviles

Rendimiento:

  1. Cachear datos cuando sea posible
  2. reactive() para compartir cálculos
  3. Limitar actualizaciones automáticas
  4. Optimizar gráficos (menos puntos si es necesario)
  5. Cargar datos fuera del server cuando puedas
